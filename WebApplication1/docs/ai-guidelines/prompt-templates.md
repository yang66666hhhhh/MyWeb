# AI 提示词模板

## 概述

本文档提供了与 AI 助手交互时使用的标准化提示词模板，帮助获得更高质量的代码输出。

---

### 代码注释

```
## 任务描述
为 [文件名] 添加代码注释。

## 注释要求
- 公共 API 必须有 XML 注释
- 复杂逻辑需要行内注释
- 解释为什么这样做，而不是做了什么

## 注释格式
```csharp
/// <summary>
/// [方法描述]
/// </summary>
/// <param name="[参数名]">[参数说明]</param>
/// <returns>[返回值说明]</returns>
```

## 输出要求
添加完整注释的代码
```

## 使用技巧

### 提高 AI 输出质量

1. **提供详细上下文**
   - 描述项目背景
   - 指定技术栈
   - 说明约束条件

2. **分步骤请求**
   - 不要一次请求太多
   - 先请求基础结构
   - 逐步添加功能

3. **提供参考示例**
   - 指定参考文件
   - 说明期望的代码风格
   - 提供类似功能的实现

4. **明确输出格式**
   - 指定文件结构
   - 说明命名规范
   - 要求注释和文档

5. **迭代改进**
   - 不满意就重新提问
   - 提供反馈改进
   - 逐步完善代码

### 常见问题

**Q: AI 生成的代码风格不一致怎么办？**
A: 在提示词中明确指定编码规范，或提供参考文件。

**Q: AI 生成的代码有 Bug 怎么办？**
A: 描述问题，提供错误信息，让 AI 重新生成。

**Q: AI 不理解业务逻辑怎么办？**
A: 提供详细的业务说明，包括流程图、示例数据等。
