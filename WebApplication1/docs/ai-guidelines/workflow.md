# AI 辅助开发工作流程

## 概述

本文档定义了使用 AI 助手进行开发的标准工作流程，确保开发过程规范、高效、可追溯。

---

## 开发流程总览

```
需求分析 → 技术设计 → 编码实现 → 代码审查 → 测试验证 → 部署上线
    ↑         ↑          ↑          ↑          ↑          ↑
    └─────────┴──────────┴──────────┴──────────┴──────────┘
                    AI 辅助各阶段
```

---

## 1. 需求分析阶段

### 输入
- 产品需求文档
- 用户故事
- 业务流程图

### AI 辅助任务
1. **需求理解**
   - 请 AI 解释复杂需求
   - 让 AI 总结关键功能点
   - 让 AI 识别潜在的技术挑战

2. **技术可行性分析**
   - 询问 AI 实现方案
   - 讨论技术选型
   - 评估开发工作量

### 输出
- 需求清单
- 技术方案初稿
- 开发任务分解

### 提示词示例
```
我需要实现一个 [功能描述]，请帮我：
1. 理解核心需求
2. 分析技术可行性
3. 建议实现方案
4. 估算开发工作量
```

---

## 2. 技术设计阶段

### 输入
- 需求清单
- 技术方案初稿
- 现有系统架构

### AI 辅助任务
1. **数据模型设计**
   - 请 AI 设计数据库表结构
   - 讨论字段类型和约束
   - 设计索引策略

2. **API 设计**
   - 请 AI 设计 RESTful API
   - 定义请求/响应格式
   - 设计错误处理机制

3. **架构设计**
   - 请 AI 设计模块结构
   - 讨论服务分层
   - 设计依赖关系

### 输出
- 数据库设计文档
- API 设计文档
- 架构设计文档

### 提示词示例
```
请为 [功能名] 设计数据模型：

## 实体信息
- 实体名称：[名称]
- 主要功能：[功能描述]
- 关联实体：[关联关系]

## 字段需求
- [字段1]: [说明]
- [字段2]: [说明]

## 输出要求
- Entity 实体类
- 关系配置
- 索引建议
```

---

## 3. 编码实现阶段

### 输入
- 技术设计文档
- API 设计文档
- 数据库设计文档

### AI 辅助任务
1. **代码生成**
   - 使用 AI 生成基础代码
   - 生成 CRUD 操作
   - 生成验证逻辑

2. **代码优化**
   - 请 AI 审查代码质量
   - 优化性能瓶颈
   - 改进代码结构

3. **问题解决**
   - 请 AI 帮助调试
   - 解决编译错误
   - 解决运行时错误

### 输出
- 完整的功能代码
- 单元测试
- 集成测试

### 工作流程

#### 步骤 1：创建实体和 DTO
```
请为 [实体名] 创建：
1. Entity 实体类
2. Create[Entity]Dto
3. Update[Entity]Dto
4. [Entity]QueryDto
5. [Entity]Dto

参考现有代码的风格。
```

#### 步骤 2：创建服务层
```
请为 [实体名] 创建服务层：
1. I[Entity]Service 接口
2. [Entity]Service 实现

功能包括：
- GetPageAsync - 分页查询
- GetByIdAsync - 获取详情
- CreateAsync - 创建
- UpdateAsync - 更新
- DeleteAsync - 删除

参考 [类似服务] 的实现。
```

#### 步骤 3：创建控制器
```
请为 [实体名] 创建控制器：
1. [Entity]Controller

端点包括：
- GET /api/[module]/[entities]
- GET /api/[module]/[entities]/{id}
- POST /api/[module]/[entities]
- PUT /api/[module]/[entities]/{id}
- DELETE /api/[module]/[entities]/{id}

参考 [类似控制器] 的实现。
```

#### 步骤 4：创建验证器
```
请为 [实体名] 创建验证器：
1. Create[Entity]DtoValidator
2. Update[Entity]DtoValidator

验证规则：
- [字段1]: [规则]
- [字段2]: [规则]
```

#### 步骤 5：创建实体配置
```
请为 [实体名] 创建实体配置：
1. [Entity]Configuration

配置包括：
- 字段类型和长度
- 必填字段
- 默认值
- 索引
```

#### 步骤 6：注册服务
```
请在 Program.cs 中注册服务：
1. 添加 using 语句
2. 注册 Service
```

#### 步骤 7：创建数据库迁移
```
请创建数据库迁移：
dotnet ef migrations add Add[EntityName]
```

## 4. 代码审查阶段

### 输入
- 完整的功能代码
- 单元测试
- 集成测试

### AI 辅助任务
1. **代码质量检查**
   - 请 AI 审查代码规范
   - 检查命名规范
   - 检查代码结构

2. **安全审查**
   - 请 AI 检查安全漏洞
   - 检查输入验证
   - 检查权限控制

3. **性能审查**
   - 请 AI 分析性能瓶颈
   - 检查 N+1 查询
   - 检查缓存策略

### 输出
- 代码审查报告
- 改进建议
- 修复后的代码

### 提示词示例
```
请审查以下代码：

## 代码文件
[粘贴代码]

## 审查重点
1. 代码规范
2. 安全性
3. 性能
4. 可维护性

## 输出要求
- 发现的问题
- 改进建议
- 修复后的代码
```

---

## 5. 测试验证阶段

### 输入
- 功能代码
- 测试需求

### AI 辅助任务
1. **单元测试编写**
   - 请 AI 生成单元测试
   - 覆盖正常流程
   - 覆盖边界条件
   - 覆盖异常情况

2. **集成测试编写**
   - 请 AI 生成集成测试
   - 测试 API 端点
   - 测试数据库操作
   - 测试业务流程

3. **测试数据准备**
   - 请 AI 生成测试数据
   - 准备种子数据
   - 准备 Mock 数据

### 输出
- 单元测试代码
- 集成测试代码
- 测试数据

### 提示词示例
```
请为 [服务名] 编写单元测试：

## 测试范围
- 方法：[方法名]
- 功能：[功能描述]

## 测试场景
1. 正常流程
2. 边界条件
3. 异常情况

## 参考测试
请参考 [测试文件] 的测试方式。

## 输出要求
- 测试类
- 测试方法（使用 xUnit）
- Mock 对象（使用 Moq）
```

## 6. 部署上线阶段

### 输入
- 测试通过的代码
- 部署配置

### AI 辅助任务
1. **部署脚本生成**
   - 请 AI 生成 Docker 配置
   - 生成 CI/CD 配置
   - 生成部署脚本

2. **文档更新**
   - 请 AI 更新 API 文档
   - 更新 README
   - 更新变更日志

3. **监控配置**
   - 请 AI 配置健康检查
   - 配置日志记录
   - 配置性能监控

### 输出
- 部署配置文件
- 更新的文档
- 监控配置

## 日常开发流程

### 新功能开发

1. **需求理解**（5分钟）
   - 阅读需求文档
   - 使用 AI 理解复杂需求

2. **技术设计**（15分钟）
   - 使用 AI 设计数据模型
   - 使用 AI 设计 API

3. **编码实现**（30-60分钟）
   - 使用 AI 生成基础代码
   - 手动调整和优化
   - 添加业务逻辑

4. **代码审查**（15分钟）
   - 使用 AI 审查代码
   - 修复发现的问题

5. **测试验证**（15分钟）
   - 使用 AI 生成测试
   - 运行测试验证

6. **文档更新**（10分钟）
   - 使用 AI 更新文档
   - 提交代码

### Bug 修复

1. **问题分析**（5分钟）
   - 使用 AI 分析错误信息
   - 定位问题原因

2. **修复方案**（5分钟）
   - 使用 AI 建议修复方案
   - 评估方案可行性

3. **代码修复**（10分钟）
   - 使用 AI 生成修复代码
   - 验证修复效果

4. **回归测试**（10分钟）
   - 使用 AI 生成回归测试
   - 运行测试验证

### 代码重构

1. **代码分析**（10分钟）
   - 使用 AI 分析现有代码
   - 识别改进点

2. **重构方案**（10分钟）
   - 使用 AI 建议重构方案
   - 评估影响范围

3. **代码重构**（20分钟）
   - 使用 AI 生成重构代码
   - 逐步重构，确保测试通过

4. **验证测试**（10分钟）
   - 运行所有测试
   - 验证功能正常

## 最佳实践

### 1. 分步骤请求
- 不要一次请求太多
- 先请求基础结构
- 逐步添加功能

### 2. 提供详细上下文
- 描述项目背景
- 指定技术栈
- 说明约束条件

### 3. 验证 AI 输出
- 检查代码正确性
- 运行测试验证
- 审查安全问题

### 4. 迭代改进
- 不满意就重新提问
- 提供反馈改进
- 逐步完善代码

### 5. 保持代码质量
- 遵循编码规范
- 编写单元测试
- 更新文档

## 常见问题

### Q: AI 生成的代码可以直接使用吗？
A: 不建议。应该：
- 仔细审查代码
- 运行测试验证
- 根据项目规范调整
- 确保安全性

### Q: 如何提高 AI 生成代码的质量？
A:
- 提供详细的上下文
- 指定代码规范
- 分步骤请求
- 提供示例参考

### Q: AI 可以帮助优化性能吗？
A: 可以，但需要：
- 提供性能指标
- 描述瓶颈所在
- 验证优化效果
- 进行性能测试

### Q: 如何处理 AI 生成的错误代码？
A:
- 描述问题和错误信息
- 提供相关代码上下文
- 让 AI 重新生成
- 手动修复并记录
