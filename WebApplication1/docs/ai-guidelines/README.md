# AI 开发规范文档

## 概述

本文档定义了使用 AI 助手进行开发时的规范和最佳实践，确保代码质量、一致性和可维护性。

## 文档结构

```
ai-guidelines/
├── README.md                 # 本文件 - AI规范概述
├── coding-standards.md       # 编码规范
├── prompt-templates.md       # 提示词模板
├── workflow.md               # 工作流程
├── code-review.md            # 代码审查清单
└── testing-guide.md          # 测试规范
```

## 核心原则

### 1. 代码质量优先
- 遵循 SOLID 原则
- 保持代码简洁、可读
- 避免重复代码
- 编写有意义的注释

### 2. 安全第一
- 永远不要在代码中硬编码敏感信息
- 始终验证用户输入
- 使用参数化查询防止 SQL 注入
- 实施适当的身份验证和授权

### 3. 测试驱动
- 为新功能编写单元测试
- 确保测试覆盖率
- 测试边界条件和异常情况

### 4. 文档完善
- 为公共 API 编写文档
- 更新 README 和变更日志
- 记录重要的设计决策

## AI 助手使用指南

### 何时使用 AI 助手

✅ **适合使用 AI 的场景：**
- 生成样板代码
- 编写单元测试
- 代码重构建议
- 文档生成
- 代码审查

⚠️ **需要谨慎的场景：**
- 安全相关的代码
- 数据库迁移
- 生产环境配置
- 性能关键代码

❌ **不应该完全依赖 AI 的场景：**
- 架构设计决策
- 业务逻辑理解
- 安全漏洞修复
- 数据模型设计

### 使用 AI 的最佳实践

1. **提供清晰的上下文**
   - 描述项目背景
   - 指定技术栈
   - 说明约束条件

2. **验证 AI 输出**
   - 检查代码正确性
   - 运行测试验证
   - 审查安全问题

3. **迭代改进**
   - 不满意就重新提问
   - 提供反馈改进
   - 逐步完善代码

## 快速开始

### 新功能开发

1. 阅读 `workflow.md` 了解开发流程
2. 使用 `prompt-templates.md` 中的模板与 AI 交互
3. 按照 `coding-standards.md` 编写代码
4. 使用 `testing-guide.md` 编写测试
5. 按照 `code-review.md` 进行代码审查

### 代码重构

1. 使用 AI 分析现有代码
2. 识别改进点
3. 逐步重构，确保测试通过
4. 更新相关文档

### Bug 修复

1. 描述问题和重现步骤
2. 使用 AI 分析可能的原因
3. 验证修复方案
4. 编写回归测试

## 常见问题

### Q: AI 生成的代码可以直接使用吗？
A: 不建议。应该：
- 仔细审查代码
- 运行测试验证
- 根据项目规范调整
- 确保安全性

### Q: 如何提高 AI 生成代码的质量？
A: 
- 提供详细的上下文
- 指定代码规范
- 分步骤请求
- 提供示例参考

### Q: AI 可以帮助优化性能吗？
A: 可以，但需要：
- 提供性能指标
- 描述瓶颈所在
- 验证优化效果
- 进行性能测试
